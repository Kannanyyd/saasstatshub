# Roadmap

SaaSStatsHub is maintained as an early-stage open-source project. This roadmap records current priorities, not guaranteed release dates.

## Near term

- [ ] Add contract tests for WPGraphQL responses and missing-field fallbacks.
- [ ] Add automated checks for canonical URLs, metadata, schema, and sitemap entries.
- [ ] Document a minimal WordPress, WPGraphQL, and custom-field setup for local contributors.
- [ ] Add accessibility checks for navigation, search, article layouts, and calculators.
- [ ] Add source-provenance validation for structured statistics and research citations.

## Medium term

- [ ] Extract reusable article and dataset components from site-specific presentation.
- [ ] Add visual regression coverage for key desktop and mobile routes.
- [ ] Improve calculator test coverage and document the tool-extension interface.
- [ ] Add a documented preview deployment workflow for pull requests.
- [ ] Publish a versioned schema for CMS fields consumed by the frontend.

## Later exploration

- [ ] Evaluate multilingual metadata and locale-aware routing.
- [ ] Explore portable import/export formats for public research datasets.
- [ ] Provide optional adapters for non-WordPress content sources.

## Proposing roadmap work

Open a GitHub issue describing the user problem, proposed scope, alternatives, and verification plan. Large roadmap items should be split into independently reviewable pull requests.
