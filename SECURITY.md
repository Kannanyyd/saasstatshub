# Security Policy

## Supported versions

SaaSStatsHub is an early-stage project. Security fixes are applied to the latest code on the `main` branch. Older commits, forks, and third-party deployments are not supported by the maintainer.

## Reporting a vulnerability

Do not open a public GitHub issue for a suspected vulnerability.

Email `sangaypopo@gmail.com` with:

- A concise description of the issue.
- The affected route, component, or commit.
- Reproduction steps or a proof of concept.
- The potential impact.
- Any suggested mitigation.

Please avoid accessing, modifying, or downloading data that does not belong to you. Do not use automated testing that could degrade the production service.

The maintainer aims to acknowledge a complete report within five business days. Validation and remediation timelines depend on severity, reproducibility, and maintainer availability. Please allow a reasonable period for investigation before public disclosure.

## Scope

This policy covers source code in this repository and the public frontend at `saasstatshub.com`. Third-party services, WordPress plugins, Cloudflare, external source websites, and independently operated forks should be reported to their respective maintainers unless the issue is caused by this project's integration code.
